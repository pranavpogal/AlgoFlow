# Pattern Transfer Skill

Use this Skill when AlgoFlow needs to explain how a structural idea transfers between coding interview problems or select a next problem based on transfer learning.

Do not use this Skill for:

- Same-topic recommendation carousels.
- Full study planning over weeks.
- Inferring mastery from classification or exposure alone.
- Random weak-topic recommendations.
- Generic model-generated recommendations without evidence.

## Procedure

1. Classify the current problem with the Problem Intelligence taxonomy.
2. Generate candidate relationships from a bounded problem corpus.
3. Score structural relationships using pattern, subpattern, structural cues, prerequisites, difficulty, and abstraction distance.
4. Integrate learner evidence conservatively.
5. Choose a transfer type from the bounded taxonomy.
6. Explain what transfers, what changes, and what remains uncertain.
7. Record recommendation events as recommendation evidence only, not mastery evidence.

## Safety

A shown recommendation, accepted recommendation, or classified pattern is not evidence of mastery. Verified independent successful transfer is a future event type, not available in Phase 4D.
